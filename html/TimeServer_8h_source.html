13// Class for relative time
14// The osaRelativeTime class is used to manage the relationship between relative

19// Although it might seem possible to implement this class without OS dependencies
20// (e.g., by using osaGetAbsoluteTime), this is problematic on some platforms, such
21// as Windows, where it is necessary to synchronize multiple time sources to get
22// a high resolution time that is relative to an absolute time. Also, even on
23// non-Windows platforms, it would not have been as efficient because the
24// native (OS-specific) "absolute time" structure would have to converted
25// to an osaAbsoluteTime structure for every call to GetRelativeTime.
